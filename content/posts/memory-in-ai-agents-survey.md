##  1，Introduction
通过理清楚和一些重要概念的区别，来初步介绍agent memory


### 和LLM memory的关系
现在这个边界已经很模糊了，基本已经包括了llm memory

### 和rag的区别

rag本质上是单次的，静态的，主要服务于单次任务，检索完，用完就行了。
而memory是一套需要维护的系统，不断的把对话和记忆整合进一个持久的库里面。

### 和context engineering的关系
是交叉而不是重叠的关系
是人使用工作台的比方，工作台上只能摆有限的资料，所以时不时需要整理，清楚，或者添加，这就是context 工程。
而agent memory就是纯靠自己的记忆和经验。



## 2，怎么存
如何存放memory

### 2.1, 存在token里

其实就是，直接像写文章一样，把一些facts用token的形式表达出来，然后放在一个地方。
至于怎么存放，可以分为1d，2d，3d。
1d就是单纯的放在一起，彼此之间没有联系
2d就是串联，整理好后的知识，能够进行推理
3d就是金字塔结构，在2d的基础上，还会抽象概括


### 2.2， 参数化的记忆
就是把记忆存在参数里，简单的又分为internal 和external两种

internal 比较有代表性的就是知识编辑，比如说rome，先定位到这个具体的知识跟哪几个矩阵有关，然后指定的去修改

external就是外接一个适配器，可以通过像lora一样的方法，像外接一个u盘，需要的时候就接就行了，本身的参数是不会变的




## 3, 为什么需要agent memory

简单来说，可以分为3个理由

1， agent 需要记住事实，比如我让agent 帮我订机票，我不希望每次都得告诉他我的姓名，护照什么的

2，agent 需要在经验中学习，同样的订机票，他能从经验中知道，哪个软件更好用，更推荐使用哪个软件，相当于错题本，知道自己犯了哪些错

3， agent需要做复杂的事情， 做的过程中，如果打开的文件过多，或者流程过长，会昏头，所以记忆的一部分也需要把流程中重要的东西给记下来
